# AI 流式输出
- 流式输出的流程？
  - 前端 发送请求到 后端
  - 后端 收到请求后，调用 大模型 进行处理
  - 大模型 处理完成后，将结果 以 流的形式 返回给 后端
  - 后端 收到 流数据 后，将数据 编码 为 字符串 并 发送给 前端
  - 前端 收到 字符串 后，将字符串 解码 为 原始数据 并 展示给 用户

- streaming 流式输出
  llm大模型交互的用户体验优化 爽否？
  - 爽
  - 爽度体现在 模型回复的速度快，用户可以边等边看，而不是等模型回复完才看

- 开启流式输出
  添加streaming:true 开启流式输出 边生成，边返回给前端
  优化了等待时间  
  后端 负责业务逻辑，前端 负责展示

- 编码
  - 任何内容都是由二进制存储 （太低效率）
  - 对内容（非文本、网络通信）进行二进制剪裁或操作
    buffer 缓冲
  - 编码 解码
    - 编码：将内容转换为二进制数组
    - 解码：将二进制数组转换为内容
    html TextEncoder TextDecoder
    buffer的底层是什么？？
    - 缓冲区（Buffer）是一种临时存储数据的区域，用于在不同组件之间传递数据。
    - 缓冲区的底层实现通常是一个数组或字节数组，用于存储二进制数据。

  ## 架构图
  - 操作系统考点
    分配资源的最小单元 进程 process
    执行的最小单元 线程 thread
    - 浏览器是多进程架构,整个进程需要各个过程之间的配合
      - _浏览器主进程_
        - 提供用户的操作反馈
          浏览器搜索框地址栏 location 输入
        - 浏览历史*入栈*新的记录
          - 现代浏览器的历史管理
            - 前进栈 (Forward Stack)：存储你可以“前进”到的页面。
            - 后退栈 (Backward Stack)：存储你可以“后退”到的页面。
        - 请求状态loading的显示
          网络请求开始，现实loading 图标
          请求完成 loading 图标隐藏
          请求的资源交给渲染进程
        - 管理子进程 进程间通信
          Inter-Process Communication (IPC)
        - 当前已经显示或者重新刷新
          beforeunload 事件
        - 浏览器主进程负责用户交互、主进程管理和文件存储（缓存、cookie、localStorage）
      - *网络进程*是面向渲染进程和浏览器进程等提供网络下载功能

1. 首先，浏览器进程收到用户输入的url请求，浏览器进程将该url转发给网络进程

- URL 补全标准URL 带上http(s)://www.
- 如果是搜索关键词，去向默认搜索引擎URL带上关键词
- 网址 英文、中文 可读性好 DNS
  OSI 七层协议 IP 地址来通信的
  拿着域名（domain） 去查询 ip地址
  key:value DNS 是一个分布式的数据库
  - 本地DNS 缓存查找
  - 局域网查找
  - 抚州的电信
  - 中国根服务器（海底光缆）
  - 美国服务器（顶级服务器）
- 如果本地有缓存，
  text/css image/jpeg
  不用请求，直接使用缓存
- 三次握手 确保双方都具有请求发送的能力
- 发送请求行和请求头
  - 请求行 方法（GET、POST等） 路径（URL） 协议版本（HTTP/1.1）
  - 请求头 键值对 描述请求的信息
    - jwt Authorization: 用于认证用户的令牌
    - Cookie: 客户端存储的会话信息
- 发送响应头，解析响应头数据，并将数据转发给浏览器进程
  Content-type 浏览器做出不一样的响应
  eg:text/html -> 渲染进程准备接收
  text/css image 下载
  其他 直接展示 如doc,mp3
  状态码
  200 成功
  301 永久重定向
  302 临时重定向
  404 未找到
  500 服务器错误
- 浏览器接收到“提交导航”的消息后，便开始准备接收html数据，接收数据的方式是直接和网络进程建立数据管道
- 最后渲染进程会向浏览器进程 “确认提交” ，已经准备好接收和解析页面数据
- 浏览器进程接收到渲染进程 “提交文档”消息后，便开始移除之前旧的文档，然后更新浏览器进程中的页面状态，进入loading状态
- 用户发出URL 请求 到页面开始解析的整个过程，就叫导航

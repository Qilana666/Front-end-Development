# 定位

- 离开文档流（Normal Flow）？

  - display:none 隐藏元素 不占用空间 不会影响到其他元素的布局
  - position:absolute;
    文档流是 HTML 元素默认的布局方式，块级元素垂直排列，行内元素水平排列，遵循从上到下，从左到右的自然顺序
  - 元素在文档流中，离开？

    1.position:relative

  - 相对定位元素，是相对于元素在文档流中的原来位置进行定位
  - 不会离开文档流，原来在标准流的位置继续占有，后面的盒子仍然以标准流的方式对待它
  - display:none 隐藏元素 不占用空间 不会影响到其他元素的布局

    2.position:absolute

  - 需要一个参考校准，这个参考标准是拥有定位属性的父元素，如果容器没有，则参考 body,离他最近的 position 不为 static 的元素
  - 会离开文档流

    3.position:fixed

    - 以浏览器为定位，固定定位也会脱离文档流
    - 固定定位的盒子，不会随着浏览器的滚动而滚动

    4.position:sticky;黏连
    根据滚动的位置，在 relative 和 fixed 之间切换
    它的行为像 relative 定位，直到滚动到指定的阈值，就像 fixed 定位一样
    粘性定位的盒子在滚动到指定位置时，会固定在指定位置

    5.position:static;
    静态定位，默认属性
    元素按照正常的文档流进行定位
    用它来取消元素的定位属性
